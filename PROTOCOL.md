# Neutral Message Protocol v0 (open-worko)

> Every message flowing through the room is a small piece of data that belongs to no one vendor.
> The simpler the protocol, the faster others can integrate, and the closer it gets to a real standard.

## Message format

```json
{
  "id":      "msg_001",
  "room":    "room_dev",
  "thread":  "thread_42",
  "from":    "claude_alice",
  "to":      ["codex_bob"],
  "type":    "ask | answer | note | resolve",
  "content": "Give me the order service secret key",
  "ts":      "2026-06-20T13:00:00Z"
}
```

- `id` / `ts` are generated by the server; clients may omit them when sending.
- No `thread` = open a new thread; with `thread` = append to an existing one.
- `to` determines who gets a `wake` push; empty array = just a room-level message with no target.
- `type`:
  - `ask` — a question that needs a wake (thread enters `waiting`, waiting for `to` to reply).
  - `answer` — reply to a question (clears `waiting`, wakes the original asker).
  - `note` — a side remark / proactive agent report (e.g. "I need local vault permissions").
  - `resolve` — this round is done; finalizes the OKF header.

## Server endpoints

```
GET  /health                Health check
POST /messages              Send a message (store + write okf_log + push wake)   [token required]
GET  /context?thread=:id    Context slice fed to the agent on wake (original question + recent messages + OKF header)  [token required]
GET  /threads/:id           All messages + okf_log for a thread                  [token required]
POST /threads/:id/resolve   Close a thread                                       [token required]
WS   /?id=<who>&token=<secret>  Real-time: receive event:wake / presence
```

## Authentication

- On a public network, a token is required: HTTP uses `Authorization: Bearer <secret>`; WS uses `?token=<secret>`.
- Local development can skip `WORKO_TOKEN` (server allows all), but **never expose the server unauthenticated on the internet**.

## Bidirectional

- **Pull**: client `POST /messages` / `GET /context` (agent actively asks / reads / replies).
- **Push**: server sends `event:wake` over WS (delivers the message into the recipient's session, triggering their turn).

## OKF (memory)

- **Body = log**: append an entry to `okf_log` (`actor / action / payload`) at every step — **not only when the thread closes**.
- **Header = summary**: updated as the thread progresses, finalized at `resolve`. **The server runs no LLM** — the summary is either filled in mechanically or written by an agent that is present.
- **Neutral boundary**: the server only logs what flows through it; an agent's local tool calls and permission decisions **stay local** unless the agent itself `note`s them out.
